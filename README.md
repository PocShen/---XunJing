# XunJing

**寻径 v1.0 - 一款好用的 URL 重定向探测插件**  
**公众号：教练说安全**  
**By：教练**

XunJing 是一款面向实战场景的 Burp Suite URL 重定向探测插件，强调**被动探测、低门槛使用、真实场景命中能力**。  
它不是只适合手工构造参数的“演示型插件”，而是更适合在日常测试、SRC、认证流量审计中长期挂载使用的实战工具。

很多 URL 重定向漏洞并不会直接暴露在当前请求中，而是隐藏在：

- 登录 / 登出 / 注册流程
- OAuth / SSO / OIDC / CAS / SAML 回调链路
- 页面中的表单、链接、跳转入口
- 多跳 302 / 301 / JS / meta refresh 跳转逻辑
- 未显式出现、但可通过补参触发的重定向参数

XunJing 的目标，就是尽可能在这些真实场景里，以更自然的方式发现潜在问题，减少手工重复测试成本，降低漏报概率。

---

## 项目特点

- 面向实战，而不是只适配靶场
- 强调被动挂载使用，正常浏览流量即可触发探测
- 对登录、登出、注册、认证回调等高价值场景有专项优化
- 新手可直接使用，老手也能通过字典和模板继续扩展
- 支持 Burp Dashboard / Issues 联动，便于统一审计与管理

---

## 核心功能

### 1. 被动自动探测
XunJing 可以在 Burp 正常使用过程中自动分析流量，对可疑请求和响应发起 URL 重定向探测，无需每次都手工构造测试数据。

支持的使用方式包括：

- Proxy 被动流量
- Target 流量
- Repeater 流量
- Scanner 集成
- 右键手动扫描

---

### 2. 多位置参数检测
插件支持对多种输入位置进行探测：

- Query 参数
- Body 表单参数
- JSON 字段
- Header 目标字段
- Path 路径位置
- 页面中发现的链接、表单、隐藏入口

这意味着它不只盯着最常见的 `?redirect=` 这种形式，也会尝试发现隐藏在路径、表单、JSON 或页面结构中的重定向入口。

---

### 3. 缺失参数自动补参
很多系统虽然没有在原始请求中显式携带重定向参数，但后端仍然支持类似：

- `url`
- `redirect`
- `next`
- `return_url`
- `callback`
- `target`
- `goto`

XunJing 可以针对这类场景自动补充缺失参数进行测试，大幅提升“看起来没有重定向点、实际上可补参利用”的发现概率。

---

### 4. 高风险参数字典
插件内置了大量重定向相关高风险参数名，并支持持续扩展，能够覆盖常见的 URL 重定向参数、认证回调参数、退出跳转参数、第三方登录参数等。

例如：

- `redirect`
- `url`
- `redirect_url`
- `redirect_uri`
- `redirect_to`
- `next`
- `return_url`
- `returnTo`
- `callback`
- `callback_url`
- `dest`
- `goto`
- `jump`
- `target`
- `backurl`
- `originUrl`
- `location`
- `service`
- `RelayState`
- `post_logout_redirect_uri`
- `target_link_uri`

---

### 5. OAuth / SSO 场景专项增强
XunJing 针对实战中高频出现的认证流做了额外优化，重点关注以下场景：

- OAuth 授权跳转
- OIDC 回调
- SSO 单点登录
- CAS / SAML 跳转链
- 登录后跳转
- 登出后跳转
- 注册完成后跳转
- 回调地址校验不足

相比普通“只改参数”的插件，这种场景化优化在实战中更有价值。

---

### 6. 多跳跳转链分析
很多 URL 重定向漏洞并不是第一跳就直接出站，而是经过：

- 302 -> 302 -> 200
- 先跳登录页，再跳目标页
- JS 跳转补刀
- meta refresh 二次跳转

XunJing 支持多跳跳转链分析，帮助识别“第一跳正常、后续跳转失控”的场景。

---

### 7. 内置 Payload 与绕过变体
插件内置多种重定向测试载荷与变体，尽可能覆盖常见校验绕过场景，包括但不限于：

- `https://evil.com`
- `//evil.com`
- `///evil.com`
- `https://trusted.com@evil.com`
- URL 编码
- 双重编码
- 斜杠 / 反斜杠混用
- host 尾点
- 点号编码
- userinfo 变体
- 混合协议形式
- 低优先级特殊字符变体

这些变体有助于提升在真实目标中的绕过命中率。

---

### 8. 页面级入口发现
插件不只分析当前请求本身，也会从响应页面中继续挖掘潜在入口，例如：

- 表单 `action`
- 输入框 `name`
- 页面中链接 `href`
- 页面暴露的可疑参数名
- 认证流相关路径和入口

这对于“当前请求无明显参数，但页面本身藏着漏洞入口”的场景尤为关键。

---

### 9. Burp Dashboard / Issues 集成
发现漏洞后，插件可以将结果同步到 Burp 的 Issues 体系中，便于统一查看与管理：

- Dashboard
- Issue 面板
- 插件 Results 页

适合在长时间被动审计中集中整理漏洞结果。

---

### 10. 新手友好设置页
为了让新手也能快速上手，XunJing 的设置页提供了更友好的模式和说明，支持：

- 快速上手
- 新手推荐模式
- 平衡模式
- 强力模式
- 仅手动模式
- 自定义参数名
- 自定义 Payload 模板
- 多线程设置
- 最大测试数设置
- 白名单设置

## 为什么选择 XunJing

传统 URL 重定向探测方式常见的问题：

- 过度依赖手工构造参数
- 只关注显式 Query 参数
- 不擅长认证流与回调链路
- 不适合长时间被动挂载使用
- 新手安装后不知道怎么用

XunJing 更关注这些实际问题：

- 尽量减少漏报
- 尽量降低手工干预成本
- 尽量让被动使用本身产生价值
- 尽量提升 OAuth / SSO / 回调类场景命中率

## 安装方法

### 方式一：直接加载编译好的 Jar
在 Burp Suite 中：

1. 进入 `Extender`
2. 打开 `Extensions`
3. 点击 `Add`
4. 选择 `Extension type: Java`
5. 选择对应版本的 Jar 文件

建议根据你的运行环境选择：

- `xunjing-jdk8.jar`
- `xunjing-jdk11plus.jar`

---

## 使用方法

### 被动使用
推荐的使用方式非常简单：

1. 在 Burp 中加载插件
2. 正常通过 Proxy / Repeater / Target 浏览目标
3. 插件会自动对高风险请求和响应进行探测
4. 在 `Results`、`Dashboard`、`Issues` 中查看结果

这也是 XunJing 最核心的使用方式。

---

### 手动使用
如果你想对某个请求单独测试，也可以：

1. 在 Burp 中选中请求
2. 右键
3. 使用插件提供的扫描入口
4. 手动触发测试

---

## 推荐配置

### 新手推荐
适合绝大多数用户：

- 开启自动扫描
- 开启高风险参数优先
- 开启缺失参数补参
- 开启 Burp Issues 同步
- 开启 Stop After First Hit
- 开启 Query / Body / JSON / Path / Header 检测
- 保持默认线程数

---

### 平衡模式
适合日常测试：

- 自动扫描开启
- 适当保留补参与多位置检测
- 控制每个目标的 Payload 数量
- 保持一定命中率与噪音平衡

---

### 强力模式
适合重点目标：

- 增加 Payload 数量
- 保留缺失参数注入
- 保留路径、Header、JSON、认证流专项检测
- 提高线程数时请注意请求量与目标风险

---

## 结果说明

插件结果通常会包含：

- 命中的请求
- 命中的响应
- 触发位置
- 使用的参数 / Payload
- 跳转证据
- 多跳跳转链信息
- 是否同步到 Burp Issues

在实战中，建议结合响应头、响应体、重定向链和业务上下文一起判断风险等级与可利用性。

设计理念
XunJing 的设计目标不是把所有功能堆进去，而是强调三点：

新手看了就能用
被动挂着就有价值
在真实业务与认证流场景中更容易命中
如果一款 URL 重定向插件只能在你主动手工构造时才有用，那它的实战价值其实是有限的。
XunJing 更希望成为你在 Burp 中长期挂载、低成本持续发现问题的工具。
